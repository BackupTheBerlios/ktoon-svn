
\section{Alcances del proyecto}
{
	El proyecto aquí planteado, deberá abordar los temas referentes a la etapa de producción de animaciones, mas precisamente los aspectos concernientes con el trazado y dibujo. \\
	Para enfrentarnos a los problemas generados en esta etapa mediante la propuesta que estamos presentando, es necesario implementar un \textbf{servidor de transacciones} que gestione y sincronice la tarea de los diferentes actores involucrados en el proceso. Los datos de entrada de este servidor serán enviados por cada uno de los animadores, que trabajan en nodos distintos.
	\ds
	El servidor deberá proveer un mecanismo de persistencia donde se almacene en un solo lugar toda la información, objetos, cuadros, escenas, etcétera y los clientes podrán recuperar esta información solicitándola al servidor de manera transparente.
	\ds
	Para que el sistema funcione bien, es necesario proveer un sistema seguro de concurrencia que permita bloquear las áreas u objetos de trabajo para que otras animadores no destruyan o modifiquen la labor que se esta realizando. De esta misma manera, es necesario restringir el acceso a los documentos para evitar que cualquier persona pueda modificar, dañar o hurtar los mismos.
	\ds
	Asimismo, es necesario implementar una aplicación cliente, que será la interfaz sobre la cual el animador trabajará, esta aplicación deberá contar con \nuevo{algunas} características de dibujo \nuevo{y deberá permitir extender o crear nuevas herramientas mediante plugins. \\
	
	Las herramientas básicas que debe contener la aplicación son:
	\begin{itemize}
	\item Herramienta de trazado de lineas
	\item Herramienta para seleccionar, escalar y rotar gráficos.
	\item Modificar nodos de un gráfico elegido.
	\item Herramienta para dibujar figuras geométricas, como lineas rectas, cuadrados, y círculos.
	\item Herramienta para añadir texto a la animación.
	\item Herramienta de ampliación (Zoom)
	\end{itemize}
	
	De la misma manera, la aplicación cliente deberá soportar al menos las siguientes características de dibujo:
	\begin{itemize}
	\item Cambio de color, grosor y textura de la linea.
	\item Cambio de color y textura de rellenos.
	\item Activación y desactivación del papel cebolla sobre el área de dibujo.
	\item Visualización y pre-visualización de la animación.
	\item Hacer y rehacer acciones realizadas.
	\end{itemize}
	
	Respecto al cliente, con este proyecto de grado no se pretende construir una solución definitiva para la creación de animaciones, en principio no necesariamente se incluirán todas las herramientas para construir animaciones de gran calidad, pero si soportará su extensión, es decir, el cliente algún día podrá alcanzar la madurez necesaria para realizar animaciones de gran calidad.
	\ds
	Una tercera y más sencilla aplicación será el cliente administrador, encargado de labores como gestionar usuarios, configuración del servidor y copias de seguridad.}
	\ds
	Como el sistema deberá permitir el trabajo desde diferentes lugares, es decir, no es necesario que los animadores se concentren en un mismo lugar a trabajar, el sistema deberá proveer un mecanismo de comunicación para que los animadores puedan dialogar sobre los aspectos referentes al proyecto en curso, este mecanismo se implementara por medio de texto y no de voz.
	\ds
	\nuevo{En sintesis, este trabajo desarrollara los siguientes aspectos:}
	\begin{itemize}
	\item El diseño y desarrollo de un servidor que soporte todos los diversos proyectos que desarrollan los clientes.
	\item Proveerá interfaces seguras para su almacenamiento y manipulación, sin permitir que ocurran daños en los diversos proyectos.
	\item Se incorporará interfaces que permitan el acceso al mismo desde diferentes sitios geográficos.
	\item Se incorporaran herramientas de trabajo a nivel básico, para que se pueda seguir desarrollando hasta alcanzar un nivel alto de aplicación.
	\end{itemize}
}

\section{Objetivos}
{
	\subsection{Objetivo general}
	{
		\begin{itemize}
		\item \nuevo{Desarrollar una herramienta para producir animaciones de forma colaborativa, para contribuir a la mejora y ahorro en los tiempos de producción de animaciones.}
		\end{itemize}
	}
	
	\newpage
	\subsection{Objetivos específicos}
	{
		\begin{itemize}
		\item Desarrollar un servidor de transacciones, apoyado en un protocolo de comunicaciones, para sincronizar clientes del servidor y gestionar la persistencia del proyecto.
		\item Definir el protocolo de comunicación con el servidor, \nuevo{mediante el análisis de las posibles acciones que se pueden realizar}, para establecer un estándar a seguir.
		\item Desarrollar un editor genérico extendido de componentes gráficos que permita al animador visualizar la creación colaborativa de la animación.
		\item Desarrollar un mecanismo de comunicación o charla entre animadores, basado en el protocolo de comunicaciones, para permitir a los miembros del equipo la fácil comunicación.
		\item Analizar las características de los componentes gráficos de la aplicación, \nuevo{abstrayendo} los elementos mas importantes que serán enviados a todos los clientes del servidor.
		\item Desarrollar un sistema de persistencia remota administrada por el servidor, basada en la jerarquía del proyecto para evitar la redundancia de información entre los clientes del servidor.
		\end{itemize}
	}
	
	\newpage
	\subsection{Objetivos estratégicos}
	{
		\begin{itemize}
		\item Permitir la producción de animaciones mas rápidamente, mediante el uso de la herramienta que se producirá al terminar este proyecto, contribuyendo así a la disminución en costos de tiempo y dinero.
		\item Permitir el uso de la herramienta desde diversos sitios de trabajo, mediante el sistema distribuido que brindara este proyecto, para acortar distancias y ahorrar en costos de transporte.
		\item Identificar errores para ayudar en la toma de decisiones sobre futuros proyectos.
		\end{itemize}
	}
