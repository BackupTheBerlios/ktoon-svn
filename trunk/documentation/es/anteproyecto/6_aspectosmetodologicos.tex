\section{Aspectos Metodológicos}
{
	\subsection{Tipo de Proyecto y Justificación de su Magnitud }
	{
		Fundamentalmente el proyecto es de tipo práctico, con algunos aspectos investigativos, al final del proceso se entregara una herramienta con las funcionalidades propuestas, debido a la magnitud de los sub-productos que serán entregados son necesarias dos personas para realizar este trabajo de grado.
	}
	
	\subsection{Resultados esperados}
	{
		\begin{itemize}
		\item Una aplicación servidor que servira para gestionar y centralizar la información.
		\item Un cliente que permita realizar animaciones y pueda usar el servidor como medio de gestión.
		\item Un cliente administrador que permita gestionar diferentes aspectos del sistema.
		\item Un documento que contenga el estándar de comunicación con el servidor.
		\item Código fuente y ejecutable de las aplicaciones y bibliotecas desarrolladas en el proyecto, artefactos de análisis y de diseño generados, documentación interna.
		\end{itemize}
	}
	
	\subsection{Metodología de desarrollo}
	{
		Para el desarrollo del proyecto se usara un proceso ágil, para esto se ha elegido la metodología \hyperlink{xp}{\textit{Programación Extrema}}, en combinación con algunos artefactos propios de \hyperlink{uml}{\textit{UML}}.
		\ds
		El enfoque elegido nos permitirá definir unas características básicas de versiones a corto plazo, con la idea de implementar y corregir un conjunto de pequeños avances.
	}
	
	\subsection{Plan de Actividades}
	{
		La \hyperlink{xp}{Programación Extrema} se compone de 4 fases importantes, que son: \textbf{plan, diseño, codificación y pruebas}, pero estas fases no se realizan de forma completa y en orden, sino que cada una se compone de actividades que se van desarrollando de manera iterativa e incremental.
		
		\subsubsection{Fase de planeación}
		{
			\begin{itemize}
			\item Escribir el documento de anteproyecto de grado.
			\item Cada iteración empieza con un plan, donde se escriben las tareas que cada iteración debe cumplir.
			\item Analisis de riesgos.
			\end{itemize}
		}
		\subsubsection{Fase de diseño}
		{
			\begin{itemize}
			\item Escribir el estándar de codificación.
			\item Escribir las políticas de uso del servidor de control de versiones.
			\item Por cada iteración, realizar \hyperlink{refactorizacion}{refactorización} cuando sea necesario.
			\item Diseño de pruebas.
			\item Definición de arquitectura y módulos.
			\end{itemize}
		}
		
		\subsubsection{Fase de codificación}
		{
			\begin{itemize}
			\item Implementación con avances rápidos y optimizaciones al final.
			\item Codificación de pruebas.
			\item Documentación de usuario.
			\item Documentación interna.
			\end{itemize}
		}
		
		\subsubsection{Fase de pruebas}
		{
			\begin{itemize}
			\item Aplicación de pruebas unitarias.
			\item Cuando se encuentra un error de codificación o de lógica, se desarrollan pruebas para evitar volver a caer en el mismo.
			\end{itemize}
		}
		
		
		La documentación de la tesis será una actividad que se desarrollara a lo largo de todo el proyecto y paralelo a las demás actividades.
	}
}
