\section{Aspectos Metodológicos}
{
	\subsection{Tipo de Proyecto y Justificación de su Magnitud }
	{
		Fundamentalmente el proyecto es de tipo practico, con algunos aspectos investigativos, al final del proceso se entregara una herramienta con las funcionalidades propuestas, debido a la magnitud de los sub-productos que serán entregados son necesarias dos personas para realizar este trabajo de grado.
	}
	
	\subsection{Resultados esperados}
	{
		\begin{itemize}
		\item Una aplicación servidor que servira para gestionar y centralizar la información.
		\item Una extensión de la aplicación \hyperlink{ktoon}{KToon} que use el servidor como medio de gestión.
		\item Un documento que contenga el estándar de comunicación con el servidor.
		\end{itemize}
	}
	
	\subsection{Metodologia de desarrollo}
	{
		Para el desarrollo del proyecto se usara un proceso ágil combinado con otras tecnicas, para esto se ha elegido la metodologia \hyperlink{xp}{\textit{Programación Extrema}}, en combinación con algunos artefactos propios de \hyperlink{uml}{\textit{UML}}.
		
		El enfoque elegido nos permitira definir unas caracteristicas basicas de versiones a corto plazo, con la idea de implementar y corregir un conjunto de pequeños avances.
	}
